# Database

Здесь размещаются материалы по базе данных и интеграционным сущностям.

## Базовые файлы

- `schema.sql` - стартовая схема `PostgreSQL`;
- будущие миграции;
- справочники соответствий;
- логи обмена;
- технические таблицы интеграции.

## Основные таблицы

- `customers` - локальный профиль клиента и связка с `Maxma`;
- `products` - карточка товара по артикулу;
- `product_images` - отдельное хранение изображений;
- `orders` - заказ и итоговые суммы;
- `order_items` - состав заказа;
- `integrations_log` - журнал запросов и ответов внешних систем;
- `sync_queue` - очередь синхронизации между `1С`, `Tilda`, `Maxma` и backend.

## Принцип хранения

- телефон является основным идентификатором клиента;
- артикул (`sku`) является основным идентификатором товара;
- итоговые суммы заказа должны фиксироваться после серверного расчета;
- все интеграционные обмены должны логироваться;
- любые повторные отправки должны идти через очередь.
