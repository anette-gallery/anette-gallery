# Что нужно от клиента для подключения всех связок

## Общий принцип

Сейчас проект подготовлен так, что:

- `frontend` можно держать на `Vercel`;
- `backend` лучше размещать отдельно от `Vercel`, например на `Railway`, `Render`, `Yandex Cloud`, `Selectel` или `VPS`;
- внешние ключи и секреты должны храниться в переменных окружения;
- до получения реальных доступов backend может работать в режиме `stub`.

## Что нужно от тебя в первую очередь

### 1. Maxma

Нужно получить:

- базовый URL API `Maxma`;
- `API key` или `Bearer token`;
- документацию по API или примеры запросов;
- список реальных endpoint, если они отличаются от наших технических путей;
- тестовый доступ или sandbox, если он есть;
- описание логики:
  - создание клиента;
  - поиск клиента по телефону;
  - получение карты лояльности;
  - проверка промокода;
  - проверка подарочной карты;
  - создание покупки;
- webhook-секрет, если `Maxma` умеет присылать события.

### 2. Tilda

Нужно получить:

- доступ к аккаунту `Tilda` или доступ администратора к проекту;
- ID сайта или проекта;
- понимание, где именно будет checkout:
  - остается в `Tilda`;
  - выносится в `Next.js`;
- способ отправки данных из `Tilda` во внешний backend;
- API ключ `Tilda`, если будет использоваться официальный API;
- доступ к формам, корзине, Zero Block или кастомному JS, если потребуется;
- список страниц и блоков, которые нельзя ломать;
- webhook/секрет, если `Tilda` будет слать уведомления на backend.

### 3. 1С 8.3

Нужно получить:

- точную конфигурацию `1С`;
- способ интеграции:
  - HTTP API;
  - опубликованный web-сервис;
  - обмен файлами;
  - регламентная выгрузка;
- URL сервиса `1С`, если он уже опубликован;
- логин и пароль;
- доступ по VPN или белому IP, если `1С` не доступна публично;
- описание справочников и документов:
  - товары;
  - цены;
  - остатки;
  - бренды;
  - размеры;
  - цвета;
- правило, что является основным идентификатором товара;
- пример выгрузки 5-10 товаров из `1С`;
- информацию, откуда брать фото:
  - из `1С`;
  - из старого сайта `Bitrix`;
  - из отдельной папки/облака.

### 4. PostgreSQL

Нужно получить:

- строку подключения `DATABASE_URL`;
- доступы к базе или решение, где ее развернуть;
- нужно ли включать SSL;
- отдельную тестовую базу и production-базу.

### 5. GitHub

Нужно получить:

- доступ к репозиторию или приглашение в организацию;
- основную ветку разработки;
- кто будет утверждать pull request;
- нужен ли `GitHub Actions` только для проверки или еще и для деплоя.

### 6. Vercel

Нужно получить:

- доступ к проекту `Vercel` или приглашение;
- указание, какой проект использовать:
  - новый;
  - существующий;
- production domain;
- preview domain, если нужен;
- понимание, будут ли frontend env vars храниться в `Vercel`.

## Какие переменные окружения понадобятся

### Переменные для backend

Эти переменные лучше хранить там, где будет жить backend:

- `PORT`
- `CORS_ORIGIN`
- `FRONTEND_PUBLIC_URL`
- `INTEGRATIONS_MODE`
- `DATABASE_URL`
- `MAXMA_API_URL`
- `MAXMA_API_KEY`
- `MAXMA_CUSTOMER_SYNC_PATH`
- `MAXMA_LOYALTY_APPLY_PATH`
- `MAXMA_CHECKOUT_CALCULATE_PATH`
- `MAXMA_PROMO_VALIDATE_PATH`
- `MAXMA_GIFT_CARD_VALIDATE_PATH`
- `MAXMA_ORDER_CREATE_PATH`
- `ONEC_API_URL`
- `ONEC_API_LOGIN`
- `ONEC_API_PASSWORD`
- `ONEC_CATALOG_SYNC_PATH`
- `ONEC_CATALOG_BATCH_SYNC_PATH`
- `TILDA_API_URL`
- `TILDA_API_KEY`
- `TILDA_WEBHOOK_SECRET`
- `TILDA_PRODUCT_UPSERT_PATH`
- `TILDA_PRODUCTS_BATCH_UPSERT_PATH`

### Переменные для frontend на Vercel

Минимально на `Vercel` для frontend понадобится:

- `NEXT_PUBLIC_API_BASE_URL`

Пример:

```env
NEXT_PUBLIC_API_BASE_URL=https://api.your-domain.ru/api/v1
```

Если backend будет отдельно от `Vercel`, то секреты `Maxma`, `1С`, `Tilda`, `DATABASE_URL` не нужно класть в `Vercel`, их надо хранить именно в окружении backend.

## Что лучше прислать мне прямо сейчас

Чтобы я мог перейти к реальному подключению, пришли:

1. `Maxma`:
   - URL API
   - токен/ключ
   - документацию или скриншоты endpoint
2. `1С`:
   - способ интеграции
   - URL
   - логин/пароль
   - пример структуры товара
3. `Tilda`:
   - доступ к проекту
   - описание, где именно останется checkout
   - API/webhook данные, если есть
4. `Infra`:
   - `DATABASE_URL`
   - домен frontend
   - домен backend
   - доступ в `GitHub`
   - доступ в `Vercel`

## Что желательно уточнить отдельно

- где будет храниться production backend;
- будет ли тестовый стенд;
- какие реальные правила совместимости скидок действуют в `Maxma`;
- кто источник правды по ценам и остаткам: `1С`, `Tilda` или другой контур;
- нужна ли синхронизация остатков в реальном времени;
- нужно ли логировать персональные данные полностью или маскировать часть значений.
